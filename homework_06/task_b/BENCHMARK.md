# Бенчмарк: FreeShiftableArray vs Native Array

## Описание

Сравнение производительности структура на основе кольцевого буфера [`FreeShiftableArray`](solution.ts) (далее - `FSA`) и нативного `Array`.
Каждая операция измеряется **отдельно**:

| Операция    | Описание                                                                                    |
| ----------- | ------------------------------------------------------------------------------------------- |
| **push**    | Вставка N элементов в конец (массив изначально пуст)                                        |
| **pop**     | Удаление N элементов с конца (массив предварительно заполнен через `fill(0)` / цикл `push`) |
| **unshift** | Вставка N элементов в начало (массив изначально пуст)                                       |
| **shift**   | Удаление N элементов с начала (массив предварительно заполнен)                              |

### Параметры запуска

| Параметр        | Значение      |
| --------------- | ------------- |
| Node.js         | v24.13.0      |
| Итерации замера | 20            |
| Размеры данных  | 1K, 10K, 100K |
| ОС              | macOS         |

### Режимы

| Режим      | Команда                                      | Описание                                                                    |
| ---------- | -------------------------------------------- | --------------------------------------------------------------------------- |
| `jit-cold` | `node benchmark.js --mode=jit-cold`          | JIT включён, без прогрева                                                   |
| `jit-warm` | `node benchmark.js --mode=jit-warm`          | JIT включён, с прогревом (5 итераций каждой bench-функции на размере 1 000) |
| `jitless`  | `node --jitless benchmark.js --mode=jitless` | JIT отключён                                                                |

## Общие выводы

1. **`push`**: Native Array быстрее в 6–12×. V8 оптимизирует `Array.push` на уровне C++, реализация кольцевого буфера на JS не может конкурировать
2. **`pop`**: С JIT — практически паритет на больших данных (Native ×1.1). Без JIT — Native Array быстрее в ~3.6×
3. **`unshift`**: FSA **драматически быстрее** на больших данных. На 100K с JIT — **×92**, без JIT — **×39**. Нативный `unshift` — O(n), кольцевой буфер — O(1) амортизированно
4. **`shift`**: Самая показательная операция. На 100K с JIT — FSA быстрее в **×657** (1.0 ms vs 662 ms). Без JIT — **×100** (6.8 ms vs 680 ms)
5. **JIT vs Jitless**: JIT критичен для `pop`/`shift` у FSA (ускорение ×6.7–6.8). Для `push`/`unshift` — ×2.4–2.5. Для нативного `unshift`/`shift` JIT не помогает (узкое место — копирование памяти O(n)).
6. **Cold vs Warm**: На больших данных (100K) разница между прогретым и холодным JIT минимальна — V8 компилирует горячие пути за первые итерации.
